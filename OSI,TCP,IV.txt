	VTP: Virtual LAN Trunking Protocol:
	===================================

	1. L2 Protocol
	2. Cisco Prop Protocol
	3. Reduces the administrative workload of VLAN Management(Create/Delete/rename) VLAN.
	4. Sends VTP packets as  Multicast packets through Trunk ports after every 5 mins, or triggered update in case of 	   any VLAN modification.

	VTP Modes:
	1. Server		Default Mode	
		1.Can create/delete/rename vlan.
		2.Sends VLAN updates with connected Switches
		3. stores information in NVRAM
		4. synchronize VTP updates
	2. Client
		1.cannot create/delete/rename vlan
		2. Synchronize VLAN information received from connected switches.
		3. cannot store in NVRAM
		4. Forwards received VLAN information
					
	3. Transparent
		1.Can create/delete/rename vlan.
		2. doesn't syncronize LOCAL VLAN updates with connected Switches
		3. stores information in NVRAM
		4. Forwards received VLAN information to connected switches.
